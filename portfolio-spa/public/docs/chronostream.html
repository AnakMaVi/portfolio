1. Arquitectura

ChronoStream separa cómputo nativo y orquestación UI: React controla el flujo de interacción y el módulo WASM ejecuta cifrado/hash en CPU local del navegador.

Componentes

  • Módulo Rust compilado a .wasm
  • Bindings JS con wasm-bindgen
  • Componente WasmCryptor para demo en vivo
  • Documentación embebida en /docs/chronostream.html

Principios

  • Demo totalmente browser-side (sin backend)
  • Medición transparente en microsegundos
  • Validación visible para reclutador
  • Pipeline reproducible local y contenedor

2. Contrato del módulo

Funciones exportadas

  • encrypt_data(input, key): cifrado XOR y salida Base64
  • generate_hash(input): hash SHA-256 en hexadecimal

Ubicación

  • chronostream-wasm/src/lib.rs
  • chronostream-wasm/Cargo.toml
  • chronostream-wasm/pkg (artefactos generados)

5. Build y Docker

docker run --rm -v c:\htdocs\portfolio\portfolio-spa\chronostream-wasm:/work chronostream-wasm-dev

wasm-pack build --target web --release

El Dockerfile instala toolchain y genera pkg/ de forma reproducible.

6. Seguridad y límites

  • El cifrado XOR es demostrativo, no apto para producción.
  • La clave vive en memoria del navegador durante el proceso.
  • No registrar secretos en logs ni telemetría.
  • Upgrade recomendado: AES-GCM o ChaCha20-Poly1305.

7. Roadmap técnico

  • Datasets de benchmark escalables (256 KB, 1 MB, 5 MB).
  • Gráfica temporal multi-ronda WASM vs JS.
  • Offloading a Web Worker para aislar UI thread.
  • Capa criptográfica enterprise y ciclo seguro de claves.
